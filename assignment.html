3.2 Alternative CSS

The alternative css is implemented with enabling and disabling an additional css file called "alternative.css".
It is applied to each page, by default overwritten by "default.css" to make sure in the first load it won't change
the style of the website.

To use it, you can click on the button in the footer called "Printer-friendly mode". If you press the button again,
it will change back to the default css style.

What happens is that, the page-breakers will now start to work, and the background effects will be "blocked"
temporarily by toggling p5.js's draw function to draw a pure black background at each framerate. The header (navbar)
and the footer will be temporarily hidden for printing as well. When one css file is being used, the other one will be blocked.
The control between two files is also implemented by javascript.

By testing, the font size and font color in the printer-friendly mode is already clear enough to see, so I didn't change them.
