Native app development and Cross-platform app development

Native app development is developing apps using develop tools and programming languages that the platform supports. For example, Java / Kotlin for Android and Objective-C / Swift / iOS SDK for iOS. They have advantages of having full access to functionalities of the platform (GPS, camera, microphone) and high performance. However, it is a bit old. We need to dynamically release new content of our app. We want to reduce the cost and simplify team structures, so one team working on many platforms is much better than multiple teams working on different platforms. Therefore, the technology of cross-platform is becoming more and more popular.
There are many attempts of finding a good solution to solve such questions. For example, HTML5 + native (Cordova, Ionic), JS + native (React Native, Weex) and custom rendering + native (Qt mobile, Flutter). But aren't many of them succeeded as Flutter. E.g., Qt mobile fails to become a popular approach for poor mobile end support and community support.


The advantages of Flutter

Ecology:
The Flutter ecosystem is growing rapidly, with an active community and a considerable number of developers and third-party components.

Technical support:
Google is currently promoting Flutter heavily, and many of its authors come from the Chromium team. The high activity on GitHub is another indication of strong support. From Flutter's inception to the present, the frequent release of new versions shows Google's significant investment in resources, so there is no need to worry about official technical support.

Development efficiency:
One codebase can run on multiple platforms, and Flutter's hot reload feature helps developers test, build UI, add functionality, and fix errors more quickly. It achieves millisecond-level hot reloading on iOS and Android simulators or physical devices without losing state. This is fantastic, and believe me, if you're a native developer, once you've experienced the Flutter development process, you may not want to go back to native development, as few people don't complain about the slow compilation speed of native development.

First of all, let's take a look at the import statement.
import 'package:flutter/material.dart';
Material is a visual design language that is widely used. Flutter provides us with various Material-styled UI components.

Just like Java, Kotlin, C/C++, a main function is an entrance of the app. In main, we can see it calls the method runApp, which starts a Flutter app. runApp takes in a Widget parameter, in this case, the MyApp object. void main() => runApp(MyApp());
Notice the => here. It's a syntax sugar for writing a function in a single line.
